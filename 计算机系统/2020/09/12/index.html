计算机系统

第一章 绪论

操作系统的地位

  • 存储程序式计算机(冯诺依曼)
    • 主存 MAR, MDR, CR
    • CPU ALU, 控制单元(PC, IR)
    • io

操作系统的形成发展

  • 元器件

    • 电子管时代
    • 集成电路时代
    • 超大规模集成电路时代

  • 操作系统阶段

    • 操作系统初级阶段

      监督程序

      • 早期批处理

        为了解决人机矛盾

      • 脱机批处理

        为了解决 CPU 高速与 io 低速的矛盾, 但是 cpu 在输入输出是会空闲

        由主机和卫星机组成

      • 执行系统

        通道和中断

        • 通道: 负责外部设备与主存之间的信息传递
        • 中断: 就是中断, interrupt

    • 操作系统的形成

      • 多道程序设计技术

        为了解决 cpu 空闲的问题, 多个独立程序相互穿插运行, 使得 cpu 和外部设备尽可能并行操作

        • 特征: 宏观上并行, 微观上还是串行

      • 分时技术

        为了解决用户不能直接控制程序运行的问题, 实现联机操作交互对话(用户使用计算机)

        分时技术使得每个终端用户在自己的终端设备上以联机方式使用计算机, 好像自己独占一样, 分时技术就是把时间分成片然后轮流分给用户

      出现了

      • 批量操作系统–>多道程序设计
      • 分时操作系统–>分时技术
      • 实时操作系统–>生产过程
        • “实时”: 计算机对于外来信息能够在被控对象允许的截止期限内做出反应

    • 操作系统的进一步发展

      • 即, 现代操作系统

操作系统的概念

  • 操作系统的特性

    • 并发: 单机操作系统的并行性, 又叫并发性, 但是是逻辑上的并行, 物理上的串行
    • 共享
    • 不确定性(用户 xjer 操作)

  • 操作系统的资源管理能力

    对系统资源实施管理和调度, 控制和协调并发活动, 对外提供用户界面

    • 处理机管理(cpu)
    • 存储器管理(主存)
      • 存储分配和存储无关性, 存储保护, 存储扩充
    • 设备管理
      • 设备无关性, 设备分配, 设备传输控制
    • 文件系统(file system)

  • 操作系统应该解决的问题

    • 资源分配的方法和策略
      • 先来先服务
      • 显现调度
      • 时间片轮转
    • 协调并发活动
      • 锁, 上锁, 开锁
      • 信号灯, 信号灯上的 p 操作 v 操作
    • 保证数据一致性
    • 数据存取

操作系统基本类型

批量操作系统

  • 将用户提交的作业成批输入计算机, 然后由作业调度程序自动选择作业运行
  • 特点:
    • 吞吐率高
    • 作业周转时间长
    • 交互性差

分时操作系统

  • 时间片轮转, 使一台机器同时为多个终端用户服务
  • 特点
    • 并行性, 独占性, 交互性

实时操作系统

  • 能够在 deadline 内处理完毕外部信息并作出反应

  • 特点

    • 实时响应
    • 高可靠性, 安全性
    • 执行装置, 咨询装置

  • 分类

    • 实时控制(工厂)
    • 实时信息处理(检索)

  • 嵌入式操作系统(EOS)

    • 区别

个人计算机操作系统

都懂

网络操作系统

  • 网络通信模块

分布式操作系统

  • 与网络操作系统的区别
    • 计算机网络存取资源是不透明的, 分布式是透明的
    • 计算机网络合作计算需要用户分配, 分布式由系统分配
    • 分布式系统对用户完全透明

UNIX, Linux

UNIX

  • 特点
    • 核心层和使用层
    • shell
    • 层次式文件系统
    • 统一看待文件和设备
    • 可移植性

Linux

  • 基础
    • UNIX
    • Minix
    • GNU
    • POSIX(portable operating system ix)
  • 特点
    • 多用户, 多任务
  • 系统组成
    • 内核: 多进程管理, 进程调度
    • shell
    • 实用工具
